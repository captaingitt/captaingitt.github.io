*Контроль версий*, также известный как управление исходным кодом, — это практика отслеживания изменений программного кода и управления ими. Системы контроля версий — это программные инструменты, помогающие командам разработчиков управлять изменениями в исходном коде с течением времени. В свете усложнения сред разработки они помогают командам разработчиков работать быстрее и эффективнее. Системы контроля версий наиболее полезны командам DevOps, поскольку помогают сократить время разработки и увеличить количество успешных развертываний.

### Работа с текстом в файлах markdown:

Для того, чтобы шрифт стал полужирным, необходимо обрамить текст с обеих сторон двойными звёздочками (**) или двойным знаком нижнего подчёркивания. 
> Например, **вот так** или __вот так__.

Для того, чтобы шрифт стал курсивным, необходимо обрамить текст с обеих сторон одинарной звёздочкой (*) или знаком нижнего подчёркивания. 
> Например, *вот так* или _вот так_.

* Альтернативные способы выделения текста жирным или курсивом нужны для того, чтобы мы могли совмещать оба этих способа. 

> Например, _текст может быть выделен курсивом и при этом быть **полужирным**_.

Для того, чтобы текст был зачёркнутым, необходимо обрамить его с обеих сторон двйоным знаком >(~~).
> Например, ~~вот так~~

# Выделение заголовков
Для того, чтобы выделить заголовок, необходимо поставить знак # в начале заголовка х n и пробел в начале строки (0 < n < 7)
# Заголовки

## Заголовки

### Заголовки

#### Заголовки

##### Заголовки

###### Заголовки

Для того, чтобы выделить заголовок, необходимо поставить знак # в начале заголовка х n и пробел в начале строки (0 < n < 7). В зависимости от количества знаков # перед текстом, меняется размер самого текста. Чем больше знаков, тем меньше шрифт. Максимальное количество знаков - 6.

Также мы можем сочетать знак # с различными способами выделения текста. Например: 

# _*Выделили текст полужирным и курсивом*_

### Списки

Нумерованные списки обозначаются цифрами 1., 2., 3. ...
1. Это первый
2. Это второй

Ненумерованные списки обозначаются знаками (*) или (+) в начале строки. Например:

* Пункт

+ Пункт

### Цитаты

Для обозначения цитат используется знак "больше" (>). Его можно вставлять как перед строкой цитаты, так и только перед строкой параграфа. 

Например, вот так:

> Йа цитатко

Также синтаксис позволяет создавать внутренние/вложенные цитаты. Для этого необходимо использовать дополнительные знаки ">". Например:

> Йа цитатко
>> Йа тоже цитатко.
>>> Ну и я с вами.

### Ссылки

Markdown поддерживает два стиля оформления ссылок:

* Гиперссылка, с немедленным указанием адреса (внутритекстовая)

* Гиперссылка, подобная сноске.

Подразумевается, что помимо URL-адреса существует ещё текст ссылки. Он заключается в квадратные скобки. Для создания внутритекстовой гиперссылки необходимо использовать круглые скобки сразу после закрывающей квадратной. Внутри них необходимо поместить URL-адрес. В них же возможно расположить название, заключённое в кавычки, которое будет отображаться при наведении, но этот пункт не является обязательным. 

Например: [пример](http://example.com/ "Необязательная подсказка")

При создании сносной гиперссылки вместо целевого адреса используется вторая пара квадратных скобок, внутри которых помещается метка, идентификатор ссылки (id). 

[Пример][id]

Markdown также позволяет использовать неявно выраженный идентификатор (сокращённый). В этом случае метка не приводится, вместо неё текст гиперссылки используется и в качестве её имени, а вторая пара квадратных скобок остаётся пустой. Например, чтобы сделать слово "Example" гиперссылкой, ведущей на сайт http//example.com , достаточно написать:

[Example][]

и затем определить гиперссылку: 

[Example]: http://example.com/

### __Изображения__

В Markdown существует 2 способа вставки изображений в документ:

1. С помощью непосредственного указания URL-адреса изображения:

![logo](https://e7.pngegg.com/pngimages/6/669/png-clipart-computer-icons-github-logo-social-media-github-logo-social-media.png)

2. Для размещения изображения с устройства необходимо поставить ![]().
Например, вот так:
![Chester](Chester.jpg)

Важной особенностью является то, что Markdown не позволяет задать размеры изображения (ширину, высоту).

## Основные команды для работы в GIT:

git init - команда для создания нового локального репозитория.

git add - первая команда в цепочке операций, предписывающей Git «сохранить» снимок текущего состояния проекта в истории коммитов.

git commit - команда сохранения текущего состояния файлов

    -m - говорит о том, что мы сохранили, далее указываем текст, который напомнит, что именно происходит в этом файле. 

git commit -a -m - "сообщение о коммите" - команда, позволяющая объединить две комманды в одну - git add и git commit. 

git log  - команда для отображения всех изменений (перечисляет коммиты, сделанные в репозитории в обратном к хронологическому порядке).

    Добавление флага -p позволяет подробно изучить изменения, внесённые в каждый файл.

    Добавление к команде --oneline позволяет увидеть список внесённых изменений без подробного описания. 

git show - команда для демонстрации полного списка изменений, внесённых конкретным коммитом. Для этого необходимо указать идентификатор или хеш коммита. *Значение хеша уникально для каждого коммита, созданного в репозитории. 
Также можно использовать сокращённый хеш - первые 4-5 символов. 

git checkout - команда для создания ветки и последующего переключения на нее, можно выполнить команду git checkout с параметром -b. 

git branch - команда для вывода списка имеющихся веток. 

git branch lists - Команда для отображения списка веток в репозитории. Это синоним команды git branch --list . Создание новой ветки с именем ＜ветка＞ . Эта команда не выполняет переключение на эту новую ветку.

git diff - команда для отображения разницы между текущей и зафиксированной версиями файла.

git checkout master - команда для возврата на ветку.

git reset - команда для отмены изменений.

git clean - Команда для удаления мусора из рабочей директории. Это могут быть результаты сборки проекта или файлы конфликтов слияний.

git merge - Команда используется для слияния одной или нескольких веток в текущую. Затем она устанавливает указатель текущей ветки на результирующий коммит.

git mergetool - Команда вызывает внешнюю программу слияний, в случае если возникли проблемы слияния.

git stash - Команда используется для временного сохранения всех незакоммиченных изменений для очистки рабочей директории без необходимости коммитить незавершённую работу в новую ветку.

git ignore - Команда для игнорирования файла, явным образом помеченный для Git как файл, который необходимо игнорировать.

git ls-tree --full-tree -r --name-only HEAD - Команда для просмотра всех файлов, содержащихся в репозитории. 

mkdir - команда, позволяющая создать папку 

git clone адрес репозитория - команда для получения копии существующего git-репозитория.

change direct или cd C:/Users/user/my_project - команда для создания репозитория в существующем каталоге. 

git rm dirname/somefile.js - команда для удаления файлов из текущего рабочего древа (файлы удаляются и из индекса).

git mv dir1/somefile.js dir2 - команда для переименования файла или папки. Указывается источник и назначение. Источник - реально существующий файл или папка, назначение - существующая папка. 

git checkout --all - Команда, отображающая все существующие папки. 

clear - команда, позволяющая очистить терминал для удобства дальнейшей работы

## Чтобы сделать Pull request, необходимо:

Сделать fork на GitHub
Скопировать ссылку репозитория
Клонировать репозиторий в VSC при помощи команды git clone link
Внести изменения в репозиторий
Закоммитить их *git commit -a -m "описание"
Отправить изменения на GitHub при помощи команды git push
В GitHub нажать кнопку Compare and pull request
Create pull request
